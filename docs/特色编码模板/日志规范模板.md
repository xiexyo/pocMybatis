# 日志编码模板规则文件


## 1. 规则目标

在当前项目中生成日志代码时，统一遵循以下目标：

1. **日志分层明确**：Controller、Service、外部接口调用分别记录不同粒度日志。
2. **日志字段标准化**：统一输出 `traceId / requestId / customerId / accountNo / productCode / code / message / cost` 等关键字段。
3. **避免日志噪音**：不重复打印、不过度打印、敏感信息脱敏。
4. **适配银行场景**：支持账户、客户、产品、交易、风控、渠道调用等通用业务。
5. **可模板化生成**：开发人员仅需修改关键参数即可完成编码。

---

## 2. 适用范围

本规则适用于以下代码位置：

- `controller` 层：接口入口、出口、异常日志
- `service` 层：业务处理开始、关键分支、成功、失败
- `repository` / `mapper` 层：通常不主动加业务日志，除非有复杂查询或性能诊断需求
- 外部接口调用场景：银行内部服务、风控服务、客户中心、核心系统、支付网关等

---

## 3. 日志组件规范

### 3.1 必须使用的日志方式

优先使用 Lombok：

```java
import lombok.extern.slf4j.Slf4j;
```

类上增加：

```java
@Slf4j
```

---

### 3.2 日志级别使用规范

#### `info`
用于记录正常业务主流程：

- 接口接收请求
- 业务处理开始/结束
- 成功创建客户/开户成功
- 调用外部接口成功
- 关键业务状态变更

#### `warn`
用于记录可预期的业务异常或风险提示：

- 参数非法
- 客户不存在
- 产品不可售
- 一类户重复开户
- 外部接口返回业务失败
- 幂等重复请求

#### `error`
用于记录系统异常、不可预期异常：

- 数据库异常
- 空指针
- 外部接口调用超时/报错
- 系统内部处理失败

#### `debug`
仅用于开发调试、非生产核心日志：

- 复杂入参详情
- 中间变量
- SQL辅助上下文

---

## 4. 日志字段标准

所有业务日志优先按以下字段输出，采用 **key=value** 风格：

- `traceId`
- `requestId`
- `bizType`
- `customerId`
- `accountNo`
- `productCode`
- `txnId`
- `channel`
- `code`
- `message`
- `cost`
- `result`

---

## 5. 敏感信息处理规则

以下字段**禁止明文完整打印**：

- `certNo`
- `mobile`
- 银行卡号/账号全量
- 密码、验证码、Token、密钥
- 任何身份证、手机号、卡号、客户隐私数据

### 5.1 脱敏示例

#### 手机号
```java
138****5678
```

#### 证件号
```java
1101********1234
```

#### 账号
```java
6222********8888
```

## 6. 当前项目推荐日志埋点位置

### 6.1 Controller 层

每个接口至少记录：

1. 接口接收请求
2. 接口成功返回
3. 接口业务异常
4. 接口系统异常（通常由全局异常兜底）

---

### 6.2 Service 层

每个核心业务方法至少记录：

1. 方法开始
2. 关键校验失败
3. 关键实体查询结果为空
4. 核心落库成功
5. 方法结束及耗时

---

### 6.3 外部接口调用

每次外部调用至少记录：

1. 调用开始
2. 调用目标系统/接口名
3. 请求关键参数
4. 响应码/响应摘要
5. 耗时
6. 失败原因

---

## 7. 日志模板占位符定义

以下占位符供模板生成时替换：

- `${ClassName}`：类名
- `${MethodName}`：方法名
- `${BizType}`：业务类型，如 `CREATE_CUSTOMER` / `OPEN_ACCOUNT`
- `${ActionName}`：动作名称，如 `创建客户` / `开户`
- `${KeyFields}`：关键业务字段，如 `customerId, productCode`
- `${RequestDTO}`：请求对象类名
- `${ResponseDTO}`：响应对象类名
- `${ExternalSystem}`：外部系统名，如 `RISK_SYSTEM`
- `${ExternalApi}`：外部接口名，如 `riskCheck`

---

## 8. Controller 日志模板

### 8.1 模板规则

- 接口入口打印简要入参
- 成功时打印结果主键和耗时
- 不在 Controller 打印完整响应对象
- 避免重复打印异常堆栈，系统异常交给全局处理器

---

### 8.2 通用模板

```java
@Slf4j
@RestController
@RequiredArgsConstructor
@RequestMapping("${BasePath}")
public class ${ClassName} {

    private final ${ServiceName} ${serviceVarName};

    @PostMapping("${SubPath}")
    public ApiResponse<${ResponseDTO}> ${MethodName}(@Valid @RequestBody ${RequestDTO} request) {
        long start = System.currentTimeMillis();
        log.info("controller start, bizType={}, action=${ActionName}, request={}", "${BizType}", request);

        ${ResponseDTO} response = ${serviceVarName}.${MethodName}(request);

        log.info("controller success, bizType={}, action=${ActionName}, result={}, cost={}ms",
                "${BizType}", response, System.currentTimeMillis() - start);
        return ApiResponse.success(response);
    }
}
```

## 9. Service 日志模板

### 9.1 模板规则

Service 是日志主战场，必须打印：

- 方法开始
- 校验失败原因
- 关键对象查询结果
- 保存结果
- 方法成功结束
- 耗时

---

### 9.2 通用模板

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class ${ClassName} implements ${ServiceInterface} {

    private final ${RepositoryA} ${repositoryAVar};
    private final ${RepositoryB} ${repositoryBVar};

    @Override
    @Transactional
    public ${ResponseDTO} ${MethodName}(${RequestDTO} request) {
        long start = System.currentTimeMillis();
        log.info("service start, bizType={}, request={}", "${BizType}", request);

        try {
            // 1. 参数/业务校验
            // 2. 查询依赖数据
            // 3. 构建实体
            // 4. 落库
            // 5. 返回结果

            log.info("service success, bizType={}, result={}, cost={}ms",
                    "${BizType}", "SUCCESS", System.currentTimeMillis() - start);

            return response;
        } catch (BizException ex) {
            log.warn("service biz fail, bizType={}, code={}, message={}, request={}, cost={}ms",
                    "${BizType}", ex.getCode(), ex.getMessage(), request, System.currentTimeMillis() - start);
            throw ex;
        } catch (Exception ex) {
            log.error("service error, bizType={}, request={}, cost={}ms",
                    "${BizType}", request, System.currentTimeMillis() - start, ex);
            throw ex;
        }
    }
}
```

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class AccountServiceImpl implements AccountService {

    private final AccountRepository accountRepository;
    private final CustomerRepository customerRepository;
    private final ProductRepository productRepository;

    @Override
    @Transactional
    public OpenAccountResponse openAccount(OpenAccountRequest request) {
        long start = System.currentTimeMillis();
        log.info("service start, bizType=OPEN_ACCOUNT, customerId={}, productCode={}, branchCode={}",
                request.getCustomerId(), request.getProductCode(), request.getBranchCode());

        try {
            CustomerEntity customer = customerRepository.findByCustomerId(request.getCustomerId());
            if (customer == null) {
                log.warn("service biz fail, bizType=OPEN_ACCOUNT, customerId={}, reason=customer_not_found",
                        request.getCustomerId());
                throw new BizException("CUST404", "客户不存在");
            }

            ProductEntity product = productRepository.findByProductCode(request.getProductCode());
            if (product == null) {
                log.warn("service biz fail, bizType=OPEN_ACCOUNT, productCode={}, reason=product_not_found",
                        request.getProductCode());
                throw new BizException("PROD404", "产品不存在");
            }

            if (!"ON_SALE".equals(product.getSaleStatus())) {
                log.warn("service biz fail, bizType=OPEN_ACCOUNT, productCode={}, saleStatus={}, reason=product_not_on_sale",
                        product.getProductCode(), product.getSaleStatus());
                throw new BizException("PROD001", "产品当前不可售");
            }

            if ("CLASS_I".equals(product.getAccountLevel())
                    && accountRepository.existsActiveClassOneAccount(customer.getCustomerId())) {
                log.warn("service biz fail, bizType=OPEN_ACCOUNT, customerId={}, productCode={}, reason=duplicate_class_one_account",
                        customer.getCustomerId(), product.getProductCode());
                throw new BizException("ACCT001", "该客户已存在一类户，不能重复开立");
            }

            LocalDateTime now = LocalDateTime.now();
            String accountNo = generateAccountNo();

            AccountEntity entity = AccountEntity.builder()
                    .accountNo(accountNo)
                    .customerId(customer.getCustomerId())
                    .accountName(customer.getCustomerName())
                    .accountType(product.getProductType())
                    .productCode(product.getProductCode())
                    .currency(product.getCurrency())
                    .balance(BigDecimal.ZERO)
                    .accountStatus("ACTIVE")
                    .branchCode(request.getBranchCode())
                    .openDate(now)
                    .createdTime(now)
                    .updatedTime(now)
                    .build();

            accountRepository.save(entity);

            log.info("service persist success, bizType=OPEN_ACCOUNT, customerId={}, accountNo={}, productCode={}, currency={}",
                    entity.getCustomerId(), maskAccountNo(entity.getAccountNo()), entity.getProductCode(), entity.getCurrency());

            OpenAccountResponse response = OpenAccountResponse.builder()
                    .accountNo(entity.getAccountNo())
                    .customerId(entity.getCustomerId())
                    .accountName(entity.getAccountName())
                    .accountType(entity.getAccountType())
                    .productCode(entity.getProductCode())
                    .currency(entity.getCurrency())
                    .balance(entity.getBalance())
                    .accountStatus(entity.getAccountStatus())
                    .branchCode(entity.getBranchCode())
                    .openDate(entity.getOpenDate())
                    .build();

            log.info("service success, bizType=OPEN_ACCOUNT, customerId={}, accountNo={}, cost={}ms",
                    response.getCustomerId(), maskAccountNo(response.getAccountNo()), System.currentTimeMillis() - start);
            return response;
        } catch (BizException ex) {
            log.warn("service biz fail, bizType=OPEN_ACCOUNT, customerId={}, productCode={}, code={}, message={}, cost={}ms",
                    request.getCustomerId(), request.getProductCode(), ex.getCode(), ex.getMessage(),
                    System.currentTimeMillis() - start);
            throw ex;
        } catch (Exception ex) {
            log.error("service error, bizType=OPEN_ACCOUNT, customerId={}, productCode={}, cost={}ms",
                    request.getCustomerId(), request.getProductCode(), System.currentTimeMillis() - start, ex);
            throw ex;
        }
    }

    private String maskAccountNo(String accountNo) {
        if (accountNo == null || accountNo.length() <= 8) {
            return accountNo;
        }
        return accountNo.substring(0, 4) + "****" + accountNo.substring(accountNo.length() - 4);
    }
```

## 11. 全局异常日志模板

你当前项目已经有 `GlobalExceptionHandler`，建议加上统一日志，避免异常无痕。

### 推荐改造模板

```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BizException.class)
    public ResponseEntity<ApiResponse<Void>> handleBizException(BizException ex) {
        log.warn("global biz exception, code={}, message={}", ex.getCode(), ex.getMessage());
        return ResponseEntity
                .status(HttpStatus.BAD_REQUEST)
                .body(ApiResponse.fail(ex.getCode(), ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ApiResponse<Void>> handleValidationException(MethodArgumentNotValidException ex) {
        FieldError fieldError = ex.getBindingResult().getFieldError();
        String message = fieldError == null ? "参数校验失败" : fieldError.getField() + ":" + fieldError.getDefaultMessage();
        log.warn("global validation exception, message={}", message);
        return ResponseEntity
                .status(HttpStatus.BAD_REQUEST)
                .body(ApiResponse.fail("P0001", message));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiResponse<Void>> handleException(Exception ex) {
        log.error("global system exception, message={}", ex.getMessage(), ex);
        return ResponseEntity
                .status(HttpStatus.INTERNAL_SERVER_ERROR)
                .body(ApiResponse.fail("S9999", ex.getMessage()));
    }
}
```

## 12. 日志生成规则摘要（供 AI 编码使用）

当为当前项目生成日志代码时，必须遵守以下规则：

### 12.1 类级规则
- 在 `Controller`、`ServiceImpl`、`RestControllerAdvice` 上添加 `@Slf4j`
- 不在 `Entity`、`DTO`、`Mapper` 接口上添加日志

### 12.2 Controller 规则
- 方法开始打印 `controller start`
- 方法成功打印 `controller success`
- 打印关键业务字段，不打印完整敏感信息
- 耗时字段统一为 `cost={}ms`

### 12.3 Service 规则
- 方法开始打印 `service start`
- 业务异常打印 `service biz fail`
- 系统异常打印 `service error`
- 落库成功打印 `service persist success`
- 成功结束打印 `service success`

### 12.4 外部调用规则
- 调用前打印 `external call start`
- 调用成功打印 `external call success`
- 业务失败打印 `external call biz fail`
- 系统失败打印 `external call error`

### 12.5 敏感数据规则
- `mobile`、`certNo`、`accountNo` 必须脱敏后打印
- 不打印密码、验证码、token、完整证件号、完整账号

### 12.6 文案规则
统一使用以下前缀之一：
- `controller start`
- `controller success`
- `service start`
- `service persist success`
- `service success`
- `service biz fail`
- `service error`
- `external call start`
- `external call success`
- `external call biz fail`
- `external call error`
- `global biz exception`
- `global validation exception`
- `global system exception`
