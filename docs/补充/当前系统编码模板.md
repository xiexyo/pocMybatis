# 零售银行 POC 工程编码规范（版本 1.0）

## 1. 适用范围
- 主语言：Java 17
- 框架：Spring Boot 3.x + MyBatis XML Mapper
- 数据库：H2（通过 schema.sql / data.sql 初始化）
- 辅语言：SQL（MyBatis XML）、YAML、Lombok

## 2. 工程架构规范

### 2.1 分层结构（强制）
```text
Controller → Service → Repository → Mapper → MyBatis XML → Database
```
- Controller：接收请求、参数校验、返回统一响应 `ApiResponse<T>`
- Service：业务逻辑、事务边界、抛出 `BizException`
- Repository：封装 Mapper，提供领域语义方法（如 `save`、`findByXxx`）
- Mapper：仅定义接口方法，无注解 SQL
- Mapper XML：存放所有 SQL
- Entity：普通 POJO，不使用 JPA 注解

### 2.2 包结构（强制）
```text
com.bank.poc
├── controller
├── service/impl
├── repository/impl
├── mapper
├── entity
├── dto/request, dto/response
├── common/api, common/exception
├── config
```
允许扩展 `client` / `client.mock`，禁止微服务、多模块、DDD、JPA/MyBatis-Plus。

## 3. Java 编码规范

### 3.1 命名规范（强制）
| 类型 | 命名模式 | 示例 |
|------|----------|------|
| Entity | `XxxEntity` | `CustomerEntity` |
| Mapper | `XxxMapper` | `CustomerMapper` |
| Mapper XML | `XxxMapper.xml` | `CustomerMapper.xml` |
| Repository 接口 | `XxxRepository` | `CustomerRepository` |
| Repository 实现 | `XxxRepositoryImpl` | `CustomerRepositoryImpl` |
| Service 接口 | `XxxService` | `CustomerService` |
| Service 实现 | `XxxServiceImpl` | `CustomerServiceImpl` |
| Request DTO | `XxxRequest` | `CreateCustomerRequest` |
| Response DTO | `XxxResponse` | `AccountDetailResponse` |
| Controller | `XxxController` | `CustomerController` |

### 3.2 返回规范（强制）
- 业务接口统一返回 `ApiResponse<T>`
- 健康检查等非业务接口可例外（如返回 `Map<String, Object>`）
- Controller 禁止直接返回 Entity

### 3.3 异常规范（强制）
- 业务异常：`throw new BizException("错误码", "错误信息");`
- 全局异常处理：`GlobalExceptionHandler`
    - `BizException` → HTTP 400，返回 `ApiResponse.fail`
    - `MethodArgumentNotValidException` → HTTP 400，错误码 `P0001`
    - 其他 `Exception` → HTTP 500，错误码 `S9999`
- 异常 message 中禁止输出敏感信息（证件号、账号等）

### 3.4 校验规范（强制）
- 使用 `jakarta.validation.*`
- 禁止使用 `javax.validation.*`、`jakarta.persistence.*`、`javax.persistence.*`

### 3.5 事务规范（强制）
- 事务注解 `@Transactional` 仅加在 Service 层方法
- Controller 禁止加事务
- XML 中不表达事务逻辑

### 3.6 金额与时间规范（强制）
- 金额：`BigDecimal`
- 时间：`LocalDateTime`
- 金额比较：`amount.compareTo(BigDecimal.ZERO)`，禁止 `==`, `>`, `<`, `equals()`

### 3.7 DTO 规范（强制）
- 请求 DTO 放 `dto.request`
- 响应 DTO 放 `dto.response`
- Service 不向外暴露 Entity

### 3.8 日志与敏感信息规范（强制）
- 不打印完整证件号、完整账号、敏感数据明文
- 不在异常 message 中暴露敏感字段

## 4. 数据访问规范（Repository + Mapper + XML）

### 4.1 通用规则（强制）
- 所有 SQL 写在 `src/main/resources/mapper/*.xml`
- Mapper 接口只定义方法，无注解 SQL
- XML `namespace` = Mapper 全限定名
- XML `id` = Mapper 方法名
- 多参数必须使用 `@Param`
- 禁止在 Java 代码中拼接 SQL
- 禁止使用 `${}` 拼接普通查询条件（防 SQL 注入）
- 动态 SQL 优先使用 `<if>`, `<where>`, `<trim>`, `<set>`
- Service 层优先通过 Repository 访问数据，禁止直接调用 Mapper

### 4.2 字段映射规范
- MyBatis 配置 `map-underscore-to-camel-case: true`
- 数据库字段 `customer_id` → Java 字段 `customerId`
- 简单查询使用 `resultType="XxxEntity"`，复杂联表可引入 `resultMap`

### 4.3 SQL 风格规范（推荐）
- SQL 关键字大写
- 字段名使用数据库原始字段名（下划线）
- 更新语句使用 `<set>` 避免遗漏逗号

## 5. 数据库规范

### 5.1 表结构变更规范（强制）
- 所有表结构增量必须同步维护 `schema.sql`
- 所有初始化/参考数据必须同步维护 `data.sql`
- 禁止仅修改 Java / XML 而不改 SQL 脚本
- 禁止依赖手工建表

### 5.2 表与字段命名规范
- 表名：`t_` + 业务名，如 `t_customer`
- 主键：业务主键（如 `customer_id`, `account_no`），无自增 ID
- 唯一索引按业务需要创建
- 外键由应用层控制，不建物理外键

### 5.3 初始数据规范
- 产品、状态枚举等参考数据通过 `data.sql` 初始化
- 示例数据清晰标注用途

## 6. 错误码规范（当前已落地）

| 分类 | 错误码 | 说明 |
|------|--------|------|
| 通用成功 | `B0000` | 成功 |
| 通用失败 | `P0001` | 参数校验失败 |
| 系统异常 | `S9999` | 系统异常 |
| 客户 | `CUST001` | 客户证件号码已存在 |
| 客户 | `CUST404` | 客户不存在 |
| 产品 | `PROD404` | 产品不存在 |
| 产品 | `PROD001` | 产品当前不可售 |
| 账户 | `ACCT001` | 该客户已存在一类户，不能重复开立 |
| 账户 | `ACCT404` | 账户不存在 |

> 后续扩展的错误码（如 `ACCT002` 余额不足、`TXN001` 交易金额非法等）应遵循相同格式，保持 6 位字符。

## 7. 状态值规范（当前已落地）

| 实体 | 字段 | 允许值 |
|------|------|--------|
| 客户 | 客户状态 | `ACTIVE` |
| 产品 | 销售状态 | `ON_SALE`, `OFF_SALE` |
| 产品 | 账户等级 | `CLASS_I`, `CLASS_II` |
| 账户 | 账户状态 | `ACTIVE` |
| 交易 | 交易类型 | `DEPOSIT`, `INTERNAL_TRANSFER` |
| 交易 | 交易状态 | `SUCCESS`, `FAILED` |
| 幂等 | 业务类型 | `DEPOSIT`, `INTERNAL_TRANSFER` |
| 幂等 | 处理状态 | `PROCESSING`, `SUCCESS`, `FAILED` |
