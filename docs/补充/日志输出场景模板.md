## 通用日志输出模板（跨系统可复用）

本文档提供一套**与业务无关**的日志规范与代码模板，适用于任何需要统一日志管理的系统（微服务、单体、批处理、网关等）。遵循本模板可确保日志的**可追溯性、可观测性、安全性**，并支持不同技术栈（Java、Go、Python 等）的概念映射。

---

### 一、核心原则

1. **结构化**：采用 `key=value` 格式，便于日志采集（ELK、Splunk、Loki）解析。
2. **分层清晰**：接入层、业务层、资源层、外部调用层职责分离，不重复打印。
3. **级别准确**：`INFO` 记录正常主流程，`WARN` 记录业务失败/降级，`ERROR` 记录系统异常。
4. **敏感脱敏**：密码、令牌、身份证、手机号、卡号等禁止明文打印。
5. **性能友好**：避免循环内打印；大对象只打印摘要；`DEBUG` 必须判断开关。
6. **链路追踪**：全链路传递 `traceId`，贯穿所有日志。

---

### 二、日志格式标准

#### 2.1 基础格式

```
level timestamp traceId={} requestId={} step={} cost={}ms code={} message={} result={} [其他业务字段...]
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `traceId` | string | 是 | 全链路追踪 ID，由入口生成或上游传入 |
| `requestId` | string | 是 | 单次请求唯一标识（同请求内不变） |
| `step` | string | 是 | 步骤标识：`start` / `success` / `fail` / `end` / `progress` |
| `cost` | int | 出口日志必填 | 耗时（毫秒） |
| `code` | string | 失败时必填 | 返回码（业务码或系统码） |
| `message` | string | 失败时必填 | 可读描述（英文或中文） |
| `result` | string | 成功时推荐 | `SUCCESS` / `FAIL` / `PARTIAL` |

#### 2.2 扩展字段（按需添加）

- `userId` / `tenantId`：租户或用户标识
- `resourceId`：操作的对象 ID（订单号、客户号、文件ID等）
- `system` / `api`：外部调用时的目标系统和接口名
- `batchId`：批量任务标识
- `retryCount`：重试次数

---

### 三、日志级别定义

| 级别 | 适用场景 | 示例 |
|------|----------|------|
| **INFO** | 正常流程关键节点：请求入口/出口、服务开始/结束、外部调用成功、状态变更、批量进度 | `step=start`，`step=success` |
| **WARN** | 业务校验失败、降级、重试耗尽、幂等拦截、可预见的异常（如记录不存在） | `step=fail, code=VALIDATION_ERROR` |
| **ERROR** | 系统异常：数据库超时、网络中断、第三方调用报错、NPE、不可恢复错误 | `step=error, code=DB_TIMEOUT` |
| **DEBUG** | 开发调试：详细入参、中间变量、SQL 参数（生产环境关闭） | `request body`，`query result` |

### 四、分层日志模板

#### 4.1 接入层（API Gateway / Controller）

**职责**：记录请求入口、出口、关键参数、耗时。  
**模板**：

```
// 入口
log.info("step=start, resource={}, method={}, clientIp={}, keyFields={}", 
         resourcePath, httpMethod, clientIp, extractKeyParams(request));

// 成功出口
log.info("step=success, resource={}, cost={}ms, resultId={}", 
         resourcePath, cost, response.getId());

// 业务失败出口
log.warn("step=fail, resource={}, cost={}ms, code={}, message={}", 
         resourcePath, cost, bizCode, bizMessage);

// 系统异常出口（交给全局处理器）
log.error("step=error, resource={}, cost={}ms", resourcePath, cost, exception);
```

#### 4.2 业务层（Service / Domain）

**职责**：记录业务流程开始/结束、分支条件、持久化结果、耗时。  
**模板**：

```
// 开始
log.info("step=start, bizType={}, keyFields={}", bizType, keyFields);

// 校验失败（WARN）
log.warn("step=fail, bizType={}, reason={}, keyFields={}", bizType, validationReason, keyFields);

// 持久化成功
log.info("step=persist, bizType={}, entityId={}", bizType, entityId);

// 业务正常结束
log.info("step=success, bizType={}, resultId={}, cost={}ms", bizType, resultId, cost);

// 业务异常（WARN）
log.warn("step=fail, bizType={}, code={}, message={}, cost={}ms", bizType, code, message, cost);

// 系统异常（ERROR）
log.error("step=error, bizType={}, cost={}ms", bizType, cost, exception);
```

#### 4.3 外部调用层（Gateway / Client）

**职责**：记录调用目标、接口、请求摘要、响应码、耗时。  
**模板**：

```
// 调用前
log.info("step=start, system={}, api={}, requestKey={}", targetSystem, apiName, requestKey);

// 调用成功
log.info("step=success, system={}, api={}, respCode={}, cost={}ms", targetSystem, apiName, respCode, cost);

// 业务失败（WARN）
log.warn("step=fail, system={}, api={}, respCode={}, message={}, cost={}ms", targetSystem, apiName, respCode, message, cost);

// 系统异常（ERROR）
log.error("step=error, system={}, api={}, cost={}ms", targetSystem, apiName, cost, exception);
```

#### 4.4 资源层（Repository / DAO）

**原则**：通常不打印日志；仅当有复杂查询或性能敏感时，使用 `DEBUG` 级别记录 SQL 摘要或耗时。

```
if (log.isDebugEnabled()) {
    log.debug("step=query, sql={}, params={}, cost={}ms", sqlSummary, params, cost);
}
```

### 五、全局异常处理模板

**职责**：统一捕获异常，打印规范日志，返回标准错误响应。

```
// 业务异常（预期内）
log.warn("step=global, type=biz, code={}, message={}", ex.getCode(), ex.getMessage());

// 参数校验异常
log.warn("step=global, type=validation, message={}", validationMessage);

// 系统异常（未预期）
log.error("step=global, type=system, message={}", ex.getMessage(), ex);
```

### 六、特殊场景日志模板

#### 6.1 批量处理

```
// 开始
log.info("step=start, batchId={}, total={}", batchId, totalCount);

// 进度（每 N 条）
log.info("step=progress, batchId={}, processed={}/{}, success={}, fail={}", 
         batchId, processed, total, successCount, failCount);

// 结束
log.info("step=end, batchId={}, total={}, success={}, fail={}, cost={}ms", 
         batchId, total, success, fail, cost);
```

#### 6.2 异步消息（MQ/Kafka）

```
// 消费开始
log.info("step=start, topic={}, partition={}, offset={}, key={}", 
         topic, partition, offset, messageKey);

// 消费成功
log.info("step=success, topic={}, key={}, cost={}ms", topic, messageKey, cost);

// 消费失败（可重试）
log.warn("step=fail, topic={}, key={}, error={}, retryCount={}", topic, messageKey, error, retry);

// 消费失败（不可重试）
log.error("step=error, topic={}, key={}, error={}", topic, messageKey, error, exception);
```

#### 6.3 定时任务

```
// 开始
log.info("step=start, task={}, schedule={}", taskName, cronExpression);

// 成功
log.info("step=success, task={}, result={}, cost={}ms", taskName, summary, cost);

// 失败
log.error("step=error, task={}, cost={}ms", taskName, cost, exception);
```

### 七、敏感信息脱敏规则

| 类型 | 脱敏规则 | 示例 |
|------|----------|------|
| 手机号 | 保留前3后4，中间4星 | `138****5678` |
| 身份证号 | 保留前4后4，中间8星 | `1101********1234` |
| 银行卡号 | 保留前4后4，中间4星 | `6222****8888` |
| 密码/验证码 | 完全隐藏，用 `***` | `password=***` |
| Token / API Key | 保留前4后4，中间8星 | `eyJh***iOiI` |
| 邮箱 | 保留前1后1域名，中间星号 | `a***@example.com` |

**脱敏工具函数示例（Java）**：
```java
public static String mask(String value, int prefix, int suffix, char maskChar) {
    if (value == null || value.length() <= prefix + suffix) return "***";
    return value.substring(0, prefix) + 
           String.valueOf(maskChar).repeat(Math.max(0, value.length() - prefix - suffix)) +
           value.substring(value.length() - suffix);
}
```

### 八、最佳实践清单

| 实践 | 说明 |
|------|------|
| ✅ 使用 MDC 传递 traceId | 日志模式配置 `%X{traceId}`，跨线程/跨服务需手动传递 |
| ✅ 避免重复打印 | 同一异常只在最上层或全局处理器打印一次 |
| ✅ 不在循环内打印 INFO | 批量任务用进度日志替代 |
| ✅ 大对象只打印摘要 | 如 `list.size()`、`JSON 前 200 字符` |
| ✅ 生产环境关闭 DEBUG | 通过配置中心动态调整日志级别 |
| ✅ 外部调用必须记录耗时 | 便于性能监控和超时分析 |
| ✅ 错误码统一 | 业务码（4-5位数字）与系统码（字母前缀）区分 |
| ✅ 日志异步输出 | 使用 `AsyncAppender` 避免阻塞主线程 |

### 九、可复用代码模板（占位符版）

#### 9.1 接入层模板

```java
// 入口
log.info("step=start, resource={}, method={}, clientIp={}, {}",
         request.getRequestURI(), request.getMethod(), getClientIp(request), extractKeys(req));

long start = System.currentTimeMillis();
try {
    resp = service.handle(req);
    log.info("step=success, resource={}, resultId={}, cost={}ms",
             request.getRequestURI(), resp.getId(), System.currentTimeMillis() - start);
    return success(resp);
} catch (BizException e) {
    log.warn("step=fail, resource={}, code={}, message={}, cost={}ms",
             request.getRequestURI(), e.getCode(), e.getMessage(), System.currentTimeMillis() - start);
    throw e;
} catch (Exception e) {
    log.error("step=error, resource={}, cost={}ms", request.getRequestURI(), System.currentTimeMillis() - start, e);
    throw e;
}
```

#### 9.2 业务层模板

```java
log.info("step=start, bizType={}, key={}", bizType, key);

long start = System.currentTimeMillis();
try {
    // 校验
    if (invalid) {
        log.warn("step=fail, bizType={}, reason={}, key={}", bizType, "validation_failed", key);
        throw new BizException("V001", "校验失败");
    }
    
    // 持久化
    String id = repository.save(entity);
    log.info("step=persist, bizType={}, entityId={}", bizType, id);
    
    log.info("step=success, bizType={}, resultId={}, cost={}ms", bizType, id, System.currentTimeMillis() - start);
    return buildResp(id);
} catch (BizException e) {
    log.warn("step=fail, bizType={}, code={}, message={}, cost={}ms", bizType, e.getCode(), e.getMessage(), System.currentTimeMillis() - start);
    throw e;
} catch (Exception e) {
    log.error("step=error, bizType={}, cost={}ms", bizType, System.currentTimeMillis() - start, e);
    throw new SystemException("SYS_ERROR");
}
```

#### 9.3 外部调用模板

```java
long start = System.currentTimeMillis();
log.info("step=start, system={}, api={}, requestKey={}", system, api, requestKey);

try {
    Resp resp = client.call(req);
    long cost = System.currentTimeMillis() - start;
    if (resp.isSuccess()) {
        log.info("step=success, system={}, api={}, respCode={}, cost={}ms", system, api, resp.getCode(), cost);
        return resp;
    } else {
        log.warn("step=fail, system={}, api={}, respCode={}, message={}, cost={}ms", system, api, resp.getCode(), resp.getMessage(), cost);
        throw new BizException(resp.getCode(), resp.getMessage());
    }
} catch (TimeoutException e) {
    log.error("step=error, system={}, api={}, reason=timeout, cost={}ms", system, api, System.currentTimeMillis() - start, e);
    throw e;
} catch (Exception e) {
    log.error("step=error, system={}, api={}, cost={}ms", system, api, System.currentTimeMillis() - start, e);
    throw e;
}
```

#### 9.4 批量处理模板

```java
log.info("step=start, batchId={}, total={}", batchId, items.size());

int success = 0, fail = 0;
for (int i = 0; i < items.size(); i++) {
    try {
        process(items.get(i));
        success++;
    } catch (Exception e) {
        fail++;
        log.warn("step=item_fail, batchId={}, index={}, key={}, error={}", batchId, i, extractKey(items.get(i)), e.getMessage());
    }
    if ((i+1) % 100 == 0) {
        log.info("step=progress, batchId={}, processed={}/{}, success={}, fail={}", batchId, i+1, items.size(), success, fail);
    }
}
log.info("step=end, batchId={}, total={}, success={}, fail={}, cost={}ms", batchId, items.size(), success, fail, System.currentTimeMillis() - start);
```

### 十、技术栈适配说明

| 技术栈 | 日志门面 | 链路追踪 | 脱敏工具 |
|--------|----------|----------|----------|
| Java | SLF4J + Logback/Log4j2 | MDC + OpenTelemetry | 自定义工具类 |
| Go | zap / logrus | context.WithValue | 自定义函数 |
| Python | logging | logging.Filter | 自定义装饰器 |
| Node.js | winston / pino | cls-hooked | 中间件 |

**关键点**：
- 无论何种语言，保持 `key=value` 输出格式不变
- 保证 `traceId` 能在异步调用、RPC 中传递
