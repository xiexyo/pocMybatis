# 客户账户分页查询

## 功能描述

该功能用于根据客户号分页查询该客户名下所有个人活期账户的列表信息。查询结果按账户创建时间倒序排列，供柜面系统、手机银行、网上银行等渠道的“账户总览”、“我的账户列表”等场景使用。

## 业务处理

1.  查询时，系统根据上送的客户号，检索该客户名下所有个人活期账户（含Ⅰ、Ⅱ、Ⅲ类户，如适用），不包括已销户账户。

2.  查询结果必须按分页方式返回，分页参数包括页码（从1开始）和每页记录数（最大不超过50条）。

3.  查询结果排序规则为：按 `created_time`（创建时间）降序，若创建时间相同则按 `account_no`（账号）降序，以保证分页结果的稳定性。

4.  查询结果中仅返回账户基本信息，不返回客户敏感信息（如证件号码、手机号码等）。返回字段包括：账号、客户号、账户名称、账户类型、产品代码、币种、余额、开户机构号、开户日期、创建时间。

5.  若客户号不存在，系统返回错误码 `CUST404`，并提示“客户不存在”。

6.  该功能仅作查询，不涉及账户状态的变更，不影响现有账户的开立、维护、销户等业务流程。

## 约束检查

1.  上送的客户号必须在核心系统中存在且客户状态正常。

2.  分页参数 `pageNo` 必须 ≥1，`pageSize` 必须在 1~50 之间，否则系统按参数校验失败处理。

3.  查询结果不包含已销户账户。

4.  若客户名下无任何账户，返回空列表，`total` 为0。

## 关键业务要素

**输入（请求）**

| 业务要素 | 业务要素说明 | 备注 |
|---|---|---|
| 客户号 | 必输，核心系统中已存在的客户唯一标识 | |
| 页码 | 必输，当前请求的页码，从1开始 | |
| 每页条数 | 必输，每页返回的最大记录数，取值范围1~50 | |

**输出（响应）**

| 业务要素 | 业务要素说明 | 备注 |
|---|---|---|
| 页码 | 当前返回的页码 | |
| 每页条数 | 每页记录数 | |
| 总记录数 | 满足条件的账户总数 | |
| 账户列表 | 当前页的账户信息数组 | 具体字段见下表 |

**账户列表项（记录）**

| 业务要素 | 业务要素说明 | 备注 |
|---|---|---|
| 账号 | 个人活期账户账号 | |
| 客户号 | 所属客户号 | |
| 账户名称 | 账户户名 | |
| 账户类型 | 如：CURRENT（活期结算）、SAVING（活期储蓄） | |
| 产品代码 | 开立时使用的产品代码 | |
| 币种 | 账户币种，如 CNY、USD | |
| 余额 | 当前账户可用余额 | |
| 开户机构号 | 账户开立机构编号 | |
| 开户日期 | 账户实际开立日期 | |
| 创建时间 | 系统记录创建时间，用于排序 | |

## 会计核算

无。

## 补充说明

- 本功能为纯查询类交易，不涉及账务处理，无需会计传票。
- 查询性能要求：在高并发场景下，单次查询响应时间应在500ms以内。数据库需保证 `customer_id` 字段有索引，避免全表扫描。
