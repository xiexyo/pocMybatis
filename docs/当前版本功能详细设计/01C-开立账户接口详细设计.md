## 2. 功能描述

开立账户接口用于为已存在客户开立账户，并根据产品配置生成账户信息，是当前账户服务主流程中的第二个核心接口。

该接口依赖：
1. 客户已存在
2. 产品已存在且可售

## 5.5 Mapper XML 设计

当前涉及的 XML 文件包括：

- `CustomerMapper.xml`
- `ProductMapper.xml`
- `AccountMapper.xml`

### AccountMapper.xml
当前至少包含以下 SQL 能力：
1. 插入账户
2. 按账号查询账户
3. 按客户号统计已存在的有效一类户数量

### 当前实现特点
- SQL 显式写在 XML 中
- Mapper 接口与 XML 一一对应
- 一类户校验通过 SQL 聚合统计完成
- 不使用 JPA Repository
- 不使用注解 SQL

## 6. 业务规则设计

## 6.1 规则一：客户必须存在
根据 `customerId` 查询客户：

```java
customerRepository.findByCustomerId(request.getCustomerId())
```

若不存在，抛出：
- 错误码：`CUST404`
- 错误信息：`客户不存在`

---

## 6.2 规则二：产品必须存在
根据 `productCode` 查询产品：

```java
productRepository.findByProductCode(request.getProductCode())
```

若不存在，抛出：
- 错误码：`PROD404`
- 错误信息：`产品不存在`

---

## 6.3 规则三：产品必须可售
当前通过产品字段 `saleStatus` 判断是否可售。

只有当：

```java
"ON_SALE".equals(product.getSaleStatus())
```

时才允许开户。

否则抛出：
- 错误码：`PROD001`
- 错误信息：`产品当前不可售`

---

## 6.4 规则四：一类户不可重复开立
当前实现逻辑如下：

1. 判断当前产品的 `accountLevel` 是否为 `CLASS_I`
2. 若不是 `CLASS_I`，则不校验该规则
3. 若是 `CLASS_I`，则通过仓储层判断客户是否已存在有效一类户
4. 仓储层调用底层 SQL，对账户表和产品表进行关联统计
5. 若统计结果大于 0，则认定该客户已存在一类户，不允许重复开立

对应仓储调用形式为：

```java
accountRepository.existsActiveClassOneAccount(customerId)
```

若违反规则，则抛出：
- 错误码：`ACCT001`
- 错误信息：`该客户已存在一类户，不能重复开立`

说明：
- 当前错误码实现以现有代码为准
- 全局约束文档中若后续统一为 `ACCT006`，则属于后续规范收敛，不属于当前已实现行为变更

## 7. 一类户限制实现设计

## 7.1 仓储方法
`AccountRepository#existsActiveClassOneAccount(String customerId)`

## 7.2 底层 Mapper 方法
`AccountMapper#countActiveClassOneAccountByCustomerId(String customerId)`

## 7.3 实现思路
1. 仓储层接收客户号
2. 调用 Mapper 统计该客户已存在的有效一类户数量
3. 底层 SQL 通过 `t_account` 与 `t_product` 进行关联
4. 过滤条件包括：
   - `a.customer_id = customerId`
   - `a.account_status = 'ACTIVE'`
   - `p.account_level = 'CLASS_I'`
5. 若统计结果大于 0，则返回已存在一类户

## 7.4 当前实现特点
- 通过 SQL 直接完成聚合判断
- 避免先查账户列表再逐条查产品的多次访问
- 查询效率优于逐条遍历校验
- 适合当前 POC 阶段

## 8. 处理流程设计

1. 客户端调用 `POST /api/v1/accounts/open`
2. Controller 接收 `OpenAccountRequest`
3. 触发参数校验
4. 调用 `accountService.openAccount(request)`
5. Service 通过 `customerRepository.findByCustomerId` 查询客户是否存在
6. Service 通过 `productRepository.findByProductCode` 查询产品是否存在
7. 校验产品是否可售
8. 若产品为一类户，则通过 `accountRepository.existsActiveClassOneAccount(customerId)` 校验是否已存在有效一类户
9. 生成账户号
10. 组装 `AccountEntity`
11. Service 调用 `accountRepository.save(entity)` 保存账户
12. Repository 调用 `accountMapper.insert(entity)` 执行落库
13. 组装 `OpenAccountResponse`
14. 返回 `ApiResponse.success(response)`

## 10. 异常处理设计

## 10.1 参数校验异常
若 `customerId` 或 `productCode` 为空，由全局异常处理器返回：
- 错误码：`P0001`

## 10.2 业务异常
当前可能抛出：

| 错误码 | 错误信息 |
|---|---|
| `CUST404` | 客户不存在 |
| `PROD404` | 产品不存在 |
| `PROD001` | 产品当前不可售 |
| `ACCT001` | 该客户已存在一类户，不能重复开立 |

## 10.3 系统异常
其他异常由 `GlobalExceptionHandler` 返回：
- 错误码：`S9999`
