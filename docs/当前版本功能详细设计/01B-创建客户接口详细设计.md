## 2. 功能描述

创建客户接口用于建立零售银行个人客户基础档案，是当前账户服务链路的起点。

后续开户功能依赖客户号，因此该接口是当前业务主流程的前置能力。

## 3.2 请求 DTO 设计

### 类名

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| customerName | String | 是 | 客户姓名 |

| mobile | String | 是 | 手机号 |

- `certNo`：`@NotBlank`
- `mobile`：`@NotBlank`

### 当前校验边界
当前仅实现基础非空校验，未实现：
- 手机号格式校验
- 证件号格式校验
- 证件类型枚举校验
- 风险等级枚举校验

### 方法

### 实现类

### 方法职责

## 5.3 Repository 设计

### 接口
`CustomerRepository`

### 当前使用的方法
```java
void save(CustomerEntity entity);

CustomerEntity findByCustomerId(String customerId);

CustomerEntity findByCertNo(String certNo);

boolean existsByCertNo(String certNo);
```

### 方法职责说明

#### existsByCertNo
用于按证件号判断客户是否已存在。

当前内部通过底层 Mapper 的统计 SQL 完成存在性判断。

#### save
用于保存客户信息到账户主数据表 `t_customer`。

#### findByCustomerId
用于按客户号查询客户信息。

说明：
- 当前创建客户接口主流程主要使用 `existsByCertNo` 和 `save`
- `findByCustomerId` 为当前工程共用仓储能力，供其他链路复用

---

## 5.4 Mapper 设计

### 接口
`CustomerMapper`

### 当前使用的方法

### 方法职责说明

## 6. 业务规则设计

## 6.1 规则一：证件号不可重复
创建客户前，先通过：

```java
customerRepository.existsByCertNo(request.getCertNo())
```

判断证件号是否已经存在。

若已存在，则抛出：
- 错误码：`CUST001`
- 错误信息：`客户证件号码已存在`

---

## 6.2 规则二：默认客户状态
当前新建客户时，客户状态固定写为：

```text
ACTIVE
```

---

## 6.3 规则三：默认风险等级
若请求中 `riskLevel` 为空或空白，则默认设置为：

```text
R2
```

若请求中已传入风险等级，则使用请求值。

当前代码逻辑等价于：

```java
request.getRiskLevel() == null || request.getRiskLevel().trim().isEmpty() ? "R2" : request.getRiskLevel()
```

---

## 6.4 规则四：自动生成 customerId
当前 `customerId` 生成规则为：

```text
C + yyyyMMddHHmmss + 4位随机数
```

示例：

```text
C202508261430001234
```

说明：
- 当前生成方式适合 POC
- 未实现分布式全局唯一生成器
- 未额外做数据库冲突重试机制

---

## 6.5 规则五：创建与更新时间同时初始化
创建客户时，系统会同时设置：

- `createdTime = now`
- `updatedTime = now`

## 7. 处理流程设计

1. 客户端调用 `POST /api/v1/customers`
2. Controller 接收 `CreateCustomerRequest`
3. 触发参数非空校验
4. 调用 `customerService.createCustomer(request)`
5. Service 调用 `customerRepository.existsByCertNo(certNo)` 校验证件号是否重复
6. 若重复，抛 `BizException("CUST001", "客户证件号码已存在")`
7. 生成客户号
8. 设置客户状态为 `ACTIVE`
9. 设置风险等级，默认 `R2`
10. 设置 `createdTime` 与 `updatedTime`
11. 组装 `CustomerEntity`
12. Service 调用 `customerRepository.save(entity)`
13. Repository 调用 `customerMapper.insert(entity)` 执行落库
14. 组装 `CreateCustomerResponse`
15. 返回 `ApiResponse.success(response)`

## 8. 事务设计

在 `CustomerServiceImpl#createCustomer` 上使用了：

```java
@Transactional
```

### 设计目的
保证创建客户落库过程具备事务性。

说明：
- 当前该接口只有单表插入
- 但仍通过事务保持写操作边界一致，符合后续扩展风格

## 9. 异常处理设计

## 9.1 参数校验异常
若请求字段为空，将由 `GlobalExceptionHandler` 捕获：
- 返回码：`P0001`

## 9.2 业务异常
若证件号已存在，将抛：
- `BizException("CUST001", "客户证件号码已存在")`

由全局异常处理器统一返回失败结构。

## 9.3 系统异常
其他未捕获异常由全局异常处理器返回：
- 错误码：`S9999`
