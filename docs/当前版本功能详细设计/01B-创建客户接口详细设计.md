# 当前已实现功能详细设计 - 创建客户接口

## 1. 文档目的

本文档用于说明当前工程中“创建客户接口”的设计与实现情况。

在阅读本文档前，请先参考：
- `docs/01-00-当前工程基线与公共设计.md`

---

## 2. 功能描述

创建客户接口用于建立零售银行个人客户基础档案，是当前账户服务链路的起点。

后续开户功能依赖客户号，因此该接口是当前业务主流程的前置能力。

## 3.2 请求 DTO 设计

### 类名
`CreateCustomerRequest`

### 字段定义
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| customerName | String | 是 | 客户姓名 |
| certType | String | 是 | 证件类型 |
| certNo | String | 是 | 证件号码 |
| mobile | String | 是 | 手机号 |
| riskLevel | String | 否 | 风险等级 |

### 参数校验
当前使用 `jakarta.validation` 进行基础非空校验：

- `customerName`：`@NotBlank`
- `certType`：`@NotBlank`
- `certNo`：`@NotBlank`
- `mobile`：`@NotBlank`

### 当前校验边界
当前仅实现基础非空校验，未实现：
- 手机号格式校验
- 证件号格式校验
- 证件类型枚举校验
- 风险等级枚举校验

### 请求示例
```json
{
  "customerName": "张三",
  "certType": "ID_CARD",
  "certNo": "110101199001011234",
  "mobile": "13800000000",
  "riskLevel": "R3"
}

## 5. 分层设计

## 5.1 Controller 设计

### 类名
`CustomerController`

### 方法
```java
@PostMapping
public ApiResponse<CreateCustomerResponse> createCustomer(@Valid @RequestBody CreateCustomerRequest request)
```

### 职责
- 接收请求参数
- 触发参数校验
- 调用 `customerService.createCustomer(request)`
- 返回统一 `ApiResponse.success(response)`

---

## 5.2 Service 设计

### 接口

### 方法

### 方法职责

## 6. 业务规则设计

## 6.1 规则一：证件号不可重复
创建客户前，先通过：
```java
customerRepository.existsByCertNo(request.getCertNo())
```
判断证件号是否已经存在。

若已存在，则抛出：
- 错误码：`CUST001`
- 错误信息：`客户证件号码已存在`

---

## 6.2 规则二：默认客户状态
当前新建客户时，客户状态固定写为：
```text
ACTIVE
```

---

## 6.3 规则三：默认风险等级
若请求中 `riskLevel` 为空或空白，则默认设置为：
```text
R2
```

若请求中已传入风险等级，则使用请求值。

---

## 6.4 规则四：自动生成 customerId
当前 customerId 生成规则为：

```text
C + yyyyMMddHHmmss + 4位随机数
```

示例：
```text
C202508261430001234
```

说明：
- 当前生成方式适合 POC
- 未实现分布式全局唯一生成器

---

## 7. 处理流程设计

1. 客户端调用 `POST /api/v1/customers`
2. Controller 接收 `CreateCustomerRequest`
3. 触发参数非空校验
4. 调用 `customerService.createCustomer(request)`
5. Service 校验证件号是否重复
6. 若重复，抛 `BizException("CUST001", "客户证件号码已存在")`
7. 生成客户号
8. 设置客户状态为 `ACTIVE`
9. 设置风险等级，默认 `R2`
10. 组装 `CustomerEntity`
11. 调用 `customerRepository.save(entity)`
12. 组装 `CreateCustomerResponse`
13. 返回 `ApiResponse.success(response)`

@Transactional
```

## 9. 异常处理设计

## 9.1 参数校验异常
若请求字段为空，将由 `GlobalExceptionHandler` 捕获：
- 返回码：`P0001`

## 9.2 业务异常
若证件号已存在，将抛：
- `BizException("CUST001", "客户证件号码已存在")`

由全局异常处理器统一返回失败结构。

## 9.3 系统异常
其他未捕获异常由全局异常处理器返回：
- 错误码：`S9999`
