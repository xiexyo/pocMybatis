## 2. 当前项目定位

当前项目是一个用于验证 AICoding 插件在银行研发场景下辅助能力的 POC 工程。

业务场景为：

**零售银行个人活期账户开立与账户服务**

当前已实现的功能包括：

1. 健康检查
2. 创建客户
3. 开立账户
4. 查询账户详情

---

## 4. 当前工程架构

项目采用标准分层架构：

```text
Controller -> Service -> Repository -> H2 Database
```

### 各层职责

#### Controller 层
- 提供 REST API
- 接收请求参数
- 调用 Service
- 返回接口响应

#### Service 层
- 承担业务逻辑
- 校验业务规则
- 调用 Repository 访问数据库

#### Repository 层
- 基于 Spring Data JPA 访问数据库
- 承担实体查询与保存

#### Entity 层
- 映射数据库表结构

#### Common / Config 层
- 提供统一返回结构
- 提供统一业务异常
- 提供全局异常处理

## 8. 当前公共能力设计

# 8.1 统一返回结构

## 类名
`ApiResponse<T>`

## 设计目的
统一业务接口响应结构，便于前后端和后续 AI 代码生成保持一致风格。

## 字段
| 字段 | 类型 | 说明 |
|---|---|---|
| code | String | 响应码 |
| message | String | 响应描述 |
| data | T | 响应数据 |

## 当前工厂方法
- `ApiResponse.success(T data)`
- `ApiResponse.success()`
- `ApiResponse.fail(String code, String message)`

## 当前成功码
- `B0000`

---

# 8.2 业务异常

## 类名
`BizException`

## 设计目的
用于在 Service 层统一表达业务错误。

## 使用方式
```java
throw new BizException("错误码", "错误信息");
```

---

# 8.3 全局异常处理

## 类名
`GlobalExceptionHandler`

## 当前处理范围

### 1）BizException
- 返回 HTTP 400
- 返回 `ApiResponse.fail(ex.getCode(), ex.getMessage())`

### 2）MethodArgumentNotValidException
- 返回 HTTP 400
- 返回码：`P0001`
- 返回信息：字段名 + 校验失败原因

### 3）Exception
- 返回 HTTP 500
- 返回码：`S9999`

---

## 9. 当前错误码基线

### 通用
- `B0000`：成功
- `P0001`：参数校验失败
- `S9999`：系统异常

### 客户类
- `CUST001`：客户证件号码已存在
- `CUST404`：客户不存在

### 产品类
- `PROD404`：产品不存在
- `PROD001`：产品当前不可售

### 账户类
- `ACCT001`：该客户已存在一类户，不能重复开立
- `ACCT404`：账户不存在

说明：
- 当前错误码以现有实现为准
- 后续扩展功能会新增更多错误码
