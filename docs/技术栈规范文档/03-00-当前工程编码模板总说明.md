1. 指导 AI 后续生成代码必须复用当前工程风格

3. 向客户或领导展示插件具备“模板沉淀与复用”能力

## 3. 当前工程基线

## 3.1 当前工程定位

当前项目是一个基于真实工程代码的零售银行账户服务 POC，当前核心实现包括：

- 创建客户
- 开立账户
- 查询账户详情
- 健康检查

当前项目目标是：

- 可运行
- 可演示
- 可继续增量开发
- 便于 AI 编码插件持续生成和修改代码

---

## 3.2 当前真实技术栈

当前工程真实技术栈如下：

- Java 17
- Spring Boot 3.x
- Spring Web
- Spring Validation
- MyBatis Spring Boot Starter
- MyBatis XML Mapper
- H2 File Database
- Lombok
- springdoc-openapi

---

## 3.3 当前真实分层结构

当前工程必须遵循以下分层结构：

```text
Controller -> Service -> Repository -> Mapper -> MyBatis XML -> H2 Database
```

各层职责如下：

- **Controller**
    - 接收请求
    - 触发参数校验
    - 返回统一响应
- **Service**
    - 编排业务逻辑
    - 做业务校验
    - 控制事务边界
    - 抛出 `BizException`
- **Repository**
    - 对 Mapper 做轻量封装
    - 向 Service 提供稳定数据访问接口
- **Mapper**
    - 仅定义数据库访问方法
- **Mapper XML**
    - 编写实际 SQL
- **Entity**
    - 仅作为数据表映射载体和查询承载对象

```text
com.bank.poc

├── service
│   └── impl

│   └── impl
```


- 引入 DDD 聚合等全新体系
- 引入 JPA / Hibernate / MyBatis-Plus

## 4. 模板整理原则

本项目模板整理必须遵循以下原则：

1. 基于当前真实工程增量提炼
2. 模板必须复用当前工程分层结构
3. 模板必须复用当前公共规范
4. 模板必须兼容当前技术栈
5. 模板必须可指导 AI 一次生成多文件代码
6. 模板必须适合增量开发，不推翻现有实现
7. 模板必须明确数据库、Java、XML 的联动关系

---

## 5. 当前工程共性规范

所有模板必须复用以下共性规范。

## 5.1 返回规范

业务接口统一返回：

```java
ApiResponse<T>
```

说明：

- 健康检查接口 `/health` 可作为非业务接口特例
- 新增业务接口不得直接返回 Entity 或裸对象

---

## 5.2 异常规范

业务异常统一使用：

```java
throw new BizException("错误码", "错误信息");
```

并由：

- `GlobalExceptionHandler`

统一处理。

说明：

- 不允许直接用 `RuntimeException` 表示业务错误
- 不允许在异常信息中暴露敏感字段

---

## 5.3 校验规范

必须使用：

```java
jakarta.validation.*
```

禁止使用：

- `javax.validation.*`
- `jakarta.persistence.*`
- `javax.persistence.*`

---

## 5.4 事务规范

事务必须定义在 Service 层，例如：

```java
@Transactional
```

说明：

- Controller 层不允许加事务
- XML 中不表达事务逻辑

---

## 5.5 数据访问规范

所有数据库访问统一走：

```text
Repository -> Mapper -> XML
```

必须遵循：

1. SQL 全部写在 `resources/mapper/*.xml`
2. Mapper 接口只定义方法
3. 多参数必须 `@Param`
4. 不允许 Java 中拼 SQL
5. 不允许注解 SQL
6. 不允许 `${}` 拼接普通查询条件

---

## 5.6 DTO 规范

- 请求 DTO 放在 `dto.request`
- 响应 DTO 放在 `dto.response`
- Controller 不直接返回 Entity
- Service 不向外暴露 Entity

---

## 5.7 金额与时间规范

- 金额统一使用 `BigDecimal`
- 时间统一使用 `LocalDateTime`

金额比较必须使用：

```java
amount.compareTo(BigDecimal.ZERO)
```

## 8. AI 使用模板的总要求

AI 在使用本项目模板生成代码时必须满足：

1. 复用当前工程真实风格
2. 保持分层结构清晰
3. 保持统一返回结构
4. 保持统一异常结构
5. 保持 Repository 层存在
6. 所有数据库访问使用 Mapper + XML
7. 不引入 JPA / Hibernate / MyBatis-Plus
8. 输出可编译、可运行代码
9. 若涉及数据库变更，必须同步修改：
    - `schema.sql`
    - `entity`
    - `mapper`
    - `mapper xml`
    - `repository`
    - `dto`
    - `service`
    - `controller`
