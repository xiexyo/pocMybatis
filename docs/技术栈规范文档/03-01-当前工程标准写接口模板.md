# 当前工程标准写接口模板

## 1. 文档目的

本文档用于基于当前真实工程代码，沉淀适用于本项目的“标准写接口模板”，指导后续 AI 或开发人员按照统一结构新增非查询类、非复杂资金类业务接口。

本模板强调：

- 复用当前工程结构
- 保持统一返回与异常风格
- 保持 Repository + Mapper + XML 数据访问模式
- 适合状态变更类、主数据写入类接口

---

## 2. 适用范围

本模板适用于以下接口类型：

- 创建客户
- 开立账户
- 账户冻结
- 账户解冻（后续扩展）
- 账户状态变更
- 非资金类标准写接口

说明：

- 当前真实工程中，创建客户、开立账户已可直接作为模板来源
- 冻结、解冻等可作为增量扩展复用本模板

## 5. 标准处理流程

标准写接口推荐遵循以下流程：

```text
参数接收
-> DTO校验
-> Service业务校验
-> Repository访问数据
-> Mapper XML执行SQL
-> Service组装Response
-> Controller返回ApiResponse
```

---

## 6. Request DTO 模板要求

## 6.1 放置位置

必须放在：

```text
com.bank.poc.dto.request
```

---

## 6.2 设计要求

1. 只承载请求参数
2. 使用 `jakarta.validation.*`
3. 字段命名保持当前工程风格
4. 不复用 Entity 作为请求对象
5. 尽量避免把数据库字段语义直接暴露为请求结构

---

## 6.3 常用注解

推荐使用：

- `@NotBlank`
- `@NotNull`
- `@Size`
- `@Pattern`

说明：

- 只做基础格式校验
- 复杂业务校验应放到 Service

---

## 6.4 示例结构

```java
@Data
public class FreezeAccountRequest {

    @NotBlank(message = "冻结原因不能为空")
    @Size(max = 255, message = "冻结原因长度不能超过255")
    private String freezeReason;

    private String operatorId;
    private String branchCode;
}
```

## 8. Controller 模板要求

## 8.1 基本要求

Controller 必须：

- 使用 `@RestController`
- 使用 `@RequestMapping`
- 使用 `@RequiredArgsConstructor`
- 使用 `@Tag` / `@Operation`
- 返回 `ApiResponse<T>`

---

## 8.2 职责边界

Controller 只负责：

1. 接收请求
2. 参数绑定
3. 触发 `@Valid` 校验
4. 调用 Service
5. 返回 `ApiResponse.success(...)`

Controller 不允许：

- 写业务逻辑
- 直接访问 Repository / Mapper
- 写 SQL
- 返回 Entity

---

## 8.3 示例模板

```java
@RestController
@RequestMapping("/api/v1/accounts")
@RequiredArgsConstructor
@Tag(name = "账户管理")
public class AccountController {

    private final AccountService accountService;

    @PostMapping("/{accountNo}/freeze")
    @Operation(summary = "冻结账户")
    public ApiResponse<FreezeAccountResponse> freezeAccount(@PathVariable String accountNo,
                                                            @Valid @RequestBody FreezeAccountRequest request) {
        return ApiResponse.success(accountService.freezeAccount(accountNo, request));
    }
}
```

## 9. Service 接口模板要求

Service 接口只定义业务能力，不写实现。

示例：

public interface AccountService {

## 10. ServiceImpl 模板要求

## 10.1 基本要求

ServiceImpl 必须：

- 使用 `@Service`
- 使用 `@RequiredArgsConstructor`
- 实现对应 Service 接口
- 在需要时使用 `@Transactional`

---

## 10.2 职责要求

ServiceImpl 负责：

1. 业务参数校验
2. 业务规则判断
3. 数据查询与写入编排
4. 抛出 `BizException`
5. 组装 Response DTO

---

## 10.3 写接口通用流程

建议按照以下步骤实现：

1. 校验主对象是否存在
2. 校验业务状态是否合法
3. 处理默认值或标准化参数
4. 调用 Repository 完成数据更新
5. 返回响应 DTO

---

## 10.4 异常要求

统一使用：

```java
throw new BizException("错误码", "错误信息");
```

示例：

```java
if (account == null) {
    throw new BizException("ACCT404", "账户不存在");
}
```

---

## 10.5 示例模板

```java
@Override
@Transactional
public FreezeAccountResponse freezeAccount(String accountNo, FreezeAccountRequest request) {
    AccountEntity account = accountRepository.findByAccountNo(accountNo);
    if (account == null) {
        throw new BizException("ACCT404", "账户不存在");
    }

    if ("FROZEN".equals(account.getAccountStatus())) {
        return FreezeAccountResponse.builder()
                .accountNo(account.getAccountNo())
                .accountStatus(account.getAccountStatus())
                .freezeReason(account.getFreezeReason())
                .updatedTime(account.getUpdatedTime())
                .build();
    }

    LocalDateTime now = LocalDateTime.now();
    accountRepository.updateStatusAndFreezeReason(accountNo, "FROZEN", request.getFreezeReason(), now);

    return FreezeAccountResponse.builder()
            .accountNo(accountNo)
            .accountStatus("FROZEN")
            .freezeReason(request.getFreezeReason())
            .updatedTime(now)
            .build();
}
```

## 11. Repository 模板要求

## 11.1 基本要求

Repository 是当前工程自定义的数据访问封装层，必须保留。

作用：

- 对 Mapper 做轻量封装
- 向 Service 提供稳定数据访问接口
- 隔离底层 XML 和 Mapper 的直接细节

---

## 11.2 示例接口

```java
public interface AccountRepository {

    AccountEntity findByAccountNo(String accountNo);

    void updateStatusAndFreezeReason(String accountNo,
                                     String accountStatus,
                                     String freezeReason,
                                     LocalDateTime updatedTime);
}
```

---

## 11.3 示例实现

```java
@Repository
@RequiredArgsConstructor
public class AccountRepositoryImpl implements AccountRepository {

    private final AccountMapper accountMapper;

    @Override
    public AccountEntity findByAccountNo(String accountNo) {
        return accountMapper.selectByAccountNo(accountNo);
    }

    @Override
    public void updateStatusAndFreezeReason(String accountNo,
                                            String accountStatus,
                                            String freezeReason,
                                            LocalDateTime updatedTime) {
        accountMapper.updateStatusAndFreezeReasonByAccountNo(accountNo, accountStatus, freezeReason, updatedTime);
    }
}
```

---

## 12. Mapper 与 XML 模板要求

## 12.1 Mapper 要求

Mapper 接口只定义方法，不写注解 SQL。

示例：

```java
public interface AccountMapper {

    AccountEntity selectByAccountNo(@Param("accountNo") String accountNo);

    int updateStatusAndFreezeReasonByAccountNo(@Param("accountNo") String accountNo,
                                               @Param("accountStatus") String accountStatus,
                                               @Param("freezeReason") String freezeReason,
                                               @Param("updatedTime") LocalDateTime updatedTime);
}
```

---

## 12.2 XML 要求

- 所有 SQL 写在 `resources/mapper/*.xml`
- `namespace` 必须与 Mapper 全限定名一致
- `id` 必须与 Mapper 方法名一致
- SQL 关键字大写

示例：

```xml
<update id="updateStatusAndFreezeReasonByAccountNo">
    UPDATE t_account
    SET account_status = #{accountStatus},
        freeze_reason = #{freezeReason},
        updated_time = #{updatedTime}
    WHERE account_no = #{accountNo}
</update>
```

---

## 13. 数据库变更模板要求

若标准写接口涉及字段新增或表结构调整，必须同步修改：

1. `schema.sql`
2. `Entity`
3. `Mapper`
4. `Mapper XML`
5. `Repository`
6. `DTO`
7. `Service`
8. `Controller`

不允许只改 Java 不改 SQL 脚本。

---

## 14. 标准错误处理模板

常见错误处理建议：

- 主对象不存在：`404` 类错误码
- 状态不合法：对应业务错误码
- 参数非法：由校验或业务异常处理

例如：

- `ACCT404`：账户不存在
- `CUST404`：客户不存在
- `PROD404`：产品不存在
- `P0001`：参数校验失败

## 15. AI 生成时必须遵循的规则

AI 使用本模板生成代码时必须：

1. 保持当前工程包结构
2. 返回 `ApiResponse<T>`
3. 使用 `BizException`
4. 使用 `jakarta.validation.*`
5. 不引入 JPA
6. 不引入注解 SQL
7. 不跳过 Repository 层
8. 输出完整代码，不只输出片段
