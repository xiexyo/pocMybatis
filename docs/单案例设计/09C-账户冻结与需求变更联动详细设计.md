# 案例C - 交易流水查询接口详细设计

## 1. 文档目的

本文档用于指导 AI 基于当前工程新增“交易流水查询接口”，用于演示银行查询类接口、动态条件查询和 MyBatis XML 动态 SQL 的开发能力。

实现本案例前，AI 必须先读取：
- `docs/09-00-全局约束与工程基线.md`

- `docs/09-02-案例B-行内转账接口详细设计.md`

---

## 2. 目标

在当前工程基础上新增交易流水查询能力，要求：

1. 支持按账户号查询交易流水
2. 支持按交易类型过滤
3. 支持按时间范围过滤
4. 支持分页查询
5. 返回查询账户视角下的交易方向与余额快照
6. 保持当前工程风格不变
7. 所有数据库访问统一通过 MyBatis Mapper + XML 动态 SQL 实现

---

## 3. 业务规则

1. `accountNo` 必填
2. 查询账户必须存在
3. `startTime`、`endTime` 若同时存在，必须满足 `startTime <= endTime`
4. 默认按 `txnTime` 倒序排序
5. 查询账户作为借方或贷方的交易都要返回
6. `direction` 规则：
   - 查询账户等于 `debitAccountNo` -> `DEBIT`
   - 查询账户等于 `creditAccountNo` -> `CREDIT`
7. `counterpartyAccountNo` 规则：
   - 当前账户为借方时，对手账号为 `creditAccountNo`
   - 当前账户为贷方时，对手账号为 `debitAccountNo`
   - 入金交易无对手账号时可为空
8. `balanceAfter` 规则：
   - 当前账户为借方时取 `debitBalanceAfter`
   - 当前账户为贷方时取 `creditBalanceAfter`
9. 默认分页参数：
   - `pageNo = 1`
   - `pageSize = 20`
10. `pageSize` 最大不超过 100

## 7.2 TransactionServiceImpl#queryTransactions 详细流程

### 方法签名
```java
public TransactionQueryResponse queryTransactions(TransactionQueryRequest request)
```

### 处理流程
1. 校验 `accountNo` 非空
2. 查询账户是否存在：
   - 不存在则抛 `BizException("ACCT404", "账户不存在")`

3. 校验时间范围：
   - 若 `startTime` 与 `endTime` 都不为空，且 `startTime.isAfter(endTime)`，抛 `BizException("TXN002", "开始时间不能大于结束时间")`

4. 处理分页参数：
   - `pageNo` 默认 1
   - `pageSize` 默认 20
   - `pageSize` 最大 100
   - `offset = (pageNo - 1) * pageSize`

5. 调用 `transactionMapper.countTransactionList(...)` 查询总条数

6. 调用 `transactionMapper.queryTransactionList(...)` 查询分页数据

7. 遍历查询结果并组装 `TransactionItemResponse`
   - 若 `accountNo.equals(txn.getDebitAccountNo())`
      - `direction = "DEBIT"`
      - `counterpartyAccountNo = txn.getCreditAccountNo()`
      - `balanceAfter = txn.getDebitBalanceAfter()`
   - 否则
      - `direction = "CREDIT"`
      - `counterpartyAccountNo = txn.getDebitAccountNo()`
      - `balanceAfter = txn.getCreditBalanceAfter()`

8. 返回 `TransactionQueryResponse`

## 8.3 TransactionMapper.xml 设计要求

### count SQL
需支持动态过滤条件：
- `accountNo`
- `txnType`
- `startTime`
- `endTime`

逻辑要求：
- `(debit_account_no = #{accountNo} OR credit_account_no = #{accountNo})`
- 时间条件动态拼接
- `txnType` 动态拼接

### list SQL
需支持：
- 与 count 同样条件
- `ORDER BY txn_time DESC`
- `LIMIT #{pageSize} OFFSET #{offset}`

### 查询字段
必须显式查询以下字段：
- `txn_id`
- `request_id`
- `debit_account_no`
- `credit_account_no`
- `txn_type`
- `txn_status`
- `amount`
- `remark`
- `txn_time`
- `created_time`
- `debit_balance_after`
- `credit_balance_after`

---

## 9. SQL 动态条件建议

建议在 XML 中使用：

```xml
<where>
    (debit_account_no = #{accountNo} OR credit_account_no = #{accountNo})
    <if test="txnType != null and txnType != ''">
        AND txn_type = #{txnType}
    </if>
    <if test="startTime != null">
        AND txn_time <![CDATA[>=]]> #{startTime}
    </if>
    <if test="endTime != null">
        AND txn_time <![CDATA[<=]]> #{endTime}
    </if>
</where>
```
