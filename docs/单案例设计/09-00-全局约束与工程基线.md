# 零售银行 AICoding POC - 全局约束与工程基线

## 1. 文档目的

本文档定义当前项目的工程基线、技术栈、架构约束、编码规范、数据库基础和统一错误码，供所有后续案例详细设计共同引用。

所有新增代码都必须基于本文档执行，禁止脱离当前工程另起一套实现。

---

## 2. 当前项目基线

## 2.1 项目定位
当前项目是一个基于 Java 17 + Spring Boot 3 + JPA + H2 的零售银行账户服务 POC 工程。

业务主线为：
- 创建客户
- 开立账户
- 查询账户详情
- 后续扩展：入金、转账、冻结、查询、幂等、Mock联调、代码治理

---

## 2.2 当前技术栈
- Java 17
- Spring Boot 3.3.2
- Spring Web
- Spring Validation
- Spring Data JPA
- H2 File Database
- Lombok
- springdoc-openapi

---

## 2.3 当前工程架构

项目采用标准分层结构：

```text
Controller -> Service -> Repository -> H2 Database
```

### 各层职责
- Controller：暴露 REST 接口，接收参数，返回统一响应
- Service：编排业务逻辑、规则校验、事务控制
- Repository：基于 Spring Data JPA 访问数据库
- Entity：数据库表结构映射
- Common/Config：公共响应、异常处理、配置

---

## 4. 全局编码规范

## 4.1 包与导包规范
必须使用：
- `jakarta.validation.*`
- `jakarta.persistence.*`

禁止使用：
- `javax.validation.*`
- `javax.persistence.*`

---

## 4.2 命名规范
- Entity：`XXXEntity`
- Repository：`XXXRepository`
- Service：`XXXService`
- Service 实现：`XXXServiceImpl`
- 请求 DTO：`XXXRequest`
- 响应 DTO：`XXXResponse`
- Controller：`XXXController`

---

## 4.3 返回规范
所有 Controller 统一返回：

```java
ApiResponse<T>
```

不得直接返回裸对象，不得各接口自行定义不同返回壳。

---

## 4.4 异常规范
业务异常统一使用：

```java
throw new BizException("错误码", "错误信息");
```

不得在 Service 中直接抛 `RuntimeException` 作为业务异常。

---

## 4.5 金额与时间规范
- 金额：`BigDecimal`
- 时间：`LocalDateTime`

金额比较必须使用：
```java
amount.compareTo(BigDecimal.ZERO)
```

---

## 4.6 事务规范
所有资金类写接口必须使用：
```java
@Transactional
```

幂等状态管理建议使用：
```java
@Transactional(propagation = Propagation.REQUIRES_NEW)
```

---

## 4.7 日志与敏感信息规范
- 不打印完整证件号
- 不打印完整账号
- 不打印敏感数据明文
- 异常信息不得暴露敏感字段

POC 阶段如未实现脱敏工具类，也不得主动输出完整敏感数据。

---

## 5. 全局错误码

### 通用
- `B0000`：成功
- `P0001`：参数校验失败
- `S9999`：系统异常

### 客户类
- `CUST404`：客户不存在
- `CUST001`：客户证件号码已存在

### 产品类
- `PROD404`：产品不存在
- `PROD001`：产品当前不可售

### 账户类
- `ACCT404`：账户不存在
- `ACCT001`：账户状态不可用
- `ACCT002`：账户余额不足
- `ACCT003`：转出账户和转入账户不能相同
- `ACCT004`：账户币种不一致
- `ACCT005`：账户已冻结
- `ACCT006`：账户已存在一类户，不能重复开立

### 交易类
- `TXN001`：交易金额必须大于0
- `TXN002`：开始时间不能大于结束时间
- `TXN003`：交易查询条件非法

### 幂等类
- `IDEMP001`：请求正在处理中
- `IDEMP002`：请求已成功处理
- `IDEMP003`：请求已失败，请更换requestId重试

### 风控类
- `RISK001`：风险校验未通过

## 8. Repository 统一增强要求

## 8.1 AccountRepository
必须支持悲观锁查询：
```java
@Lock(LockModeType.PESSIMISTIC_WRITE)
@Query("select a from AccountEntity a where a.accountNo = :accountNo")
Optional<AccountEntity> lockByAccountNo(@Param("accountNo") String accountNo);
```

## 8.2 TransactionRepository
建议继承：
```java
JpaRepository<TransactionEntity, String>, JpaSpecificationExecutor<TransactionEntity>
```

并保留：
```java
Optional<TransactionEntity> findByRequestId(String requestId);
```

## 8.3 IdempotentRecordRepository
建议新增：
```java
Optional<IdempotentRecordEntity> findByRequestIdAndBusinessType(String requestId, String businessType);
```
