## 2. 目标

在当前工程基础上实现：

1. 交易流水查询接口
2. 按 `accountNo` 查询账户相关流水
3. 支持 `txnType`、`startTime`、`endTime` 过滤
4. 支持分页
5. 按 `txnTime` 倒序
6. 返回当前账户视角的方向、对手账号和余额快照
7. 避免低质量查询实现（如全量查出后内存过滤）
8. 所有数据库访问统一通过 MyBatis Mapper + XML 动态 SQL 实现

---

## 3. 业务规则

1. `accountNo` 必填
2. `accountNo` 必须存在
3. `startTime` 和 `endTime` 若同时存在，则必须满足 `startTime <= endTime`
4. `pageNo` 默认 1
5. `pageSize` 默认 20，最大 100
6. 返回当前账户作为借方或贷方的所有交易
7. 默认按 `txnTime desc` 排序
8. 入金交易允许无对手账号
9. 返回结果中必须计算当前账户视角的：
   - `direction`
   - `counterpartyAccountNo`
   - `balanceAfter`

## 7.2 TransactionServiceImpl#queryTransactions 详细流程

### 方法签名
```java
public TransactionQueryResponse queryTransactions(TransactionQueryRequest request)
```

### 处理流程
1. 校验 `accountNo` 必填
2. 查询账户是否存在，不存在抛 `ACCT404`
3. 初始化分页参数：
   - `pageNo = request.pageNo == null || request.pageNo < 1 ? 1 : request.pageNo`
   - `pageSize = request.pageSize == null || request.pageSize < 1 ? 20 : request.pageSize`
   - 若 `pageSize > 100`，直接限制为 100
4. 校验时间范围：
   - 若 `startTime != null && endTime != null && startTime.isAfter(endTime)`，抛 `TXN002`
5. 计算分页偏移量：
   - `offset = (pageNo - 1) * pageSize`
6. 调用 `transactionMapper.countTransactionList(...)`
7. 调用 `transactionMapper.queryTransactionList(...)`
8. 将结果列表转换为 `TransactionQueryResponse`

说明：
- 不允许先 `select all` 再 Java 内存过滤
- 不允许 Java 内存分页
- 必须由数据库完成过滤、排序、分页

## 8. 结果转换规则

对每一条 `TransactionEntity`，根据当前 `accountNo` 决定返回视角：

### 8.1 若当前账户为借方
条件：
```text
accountNo == transaction.debitAccountNo
```

返回：
- `direction = "DEBIT"`
- `counterpartyAccountNo = transaction.creditAccountNo`
- `balanceAfter = transaction.debitBalanceAfter`

### 8.2 若当前账户为贷方
条件：
```text
accountNo == transaction.creditAccountNo
```

返回：
- `direction = "CREDIT"`
- `counterpartyAccountNo = transaction.debitAccountNo`
- `balanceAfter = transaction.creditBalanceAfter`

### 8.3 若是入金交易
由于 `debitAccountNo = null`，当前账户必为贷方：
- `direction = "CREDIT"`
- `counterpartyAccountNo = null`
- `balanceAfter = creditBalanceAfter`

## 10. TransactionMapper.xml 设计要求

## 10.1 count SQL
必须支持动态条件：
- `accountNo`
- `txnType`
- `startTime`
- `endTime`

SQL 逻辑：
```sql
(debit_account_no = #{accountNo} OR credit_account_no = #{accountNo})
```

并结合：
- `txn_type = #{txnType}`（当有值时）
- `txn_time >= #{startTime}`（当有值时）
- `txn_time <= #{endTime}`（当有值时）

---

## 10.2 list SQL
必须支持：
- 与 count 相同的动态过滤条件
- `ORDER BY txn_time DESC`
- `LIMIT #{pageSize} OFFSET #{offset}`

必须显式查询字段：
- `txn_id`
- `request_id`
- `debit_account_no`
- `credit_account_no`
- `txn_type`
- `txn_status`
- `amount`
- `remark`
- `txn_time`
- `created_time`
- `debit_balance_after`
- `credit_balance_after`

---

## 10.3 动态 SQL 建议
建议在 XML 中使用：

```xml
<where>
    (debit_account_no = #{accountNo} OR credit_account_no = #{accountNo})
    <if test="txnType != null and txnType != ''">
        AND txn_type = #{txnType}
    </if>
    <if test="startTime != null">
        AND txn_time <![CDATA[>=]]> #{startTime}
    </if>
    <if test="endTime != null">
        AND txn_time <![CDATA[<=]]> #{endTime}
    </if>
</where>
```
