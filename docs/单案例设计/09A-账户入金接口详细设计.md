# 案例A - 账户入金接口详细设计

## 1. 文档目的

本文档用于指导 AI 基于当前工程新增“账户入金接口”，用于演示银行资金类接口的新增开发能力。

实现本案例前，AI 必须先读取：
- `docs/09-00-全局约束与工程基线.md`

---

## 2. 目标

在当前工程基础上新增账户入金能力，要求：

1. 支持对指定账户入金
2. 更新账户余额
3. 写入交易流水
4. 支持 requestId 幂等控制
5. 返回首次成功结果重放
6. 保持当前工程风格不变

---

## 3. 业务规则

1. `requestId` 必填
2. `accountNo` 必填
3. `amount` 必须大于 0
4. 账户必须存在
5. 账户状态必须为 `ACTIVE`
6. 若账户为 `FROZEN`，返回 `ACCT005`
7. 入金成功后账户余额增加
8. 必须写入一笔 `DEPOSIT` 交易流水
9. 同一 `requestId` 重复请求不可重复入账
10. 重复请求若已成功，需返回首次成功结果

## 8.3 TransactionServiceImpl#deposit 详细流程

### 方法签名
```java
@Transactional
public DepositResponse deposit(DepositRequest request)
```

### 处理流程
1. 校验 `amount.compareTo(BigDecimal.ZERO) > 0`
    - 否则抛 `BizException("TXN001", "交易金额必须大于0")`
2. 查询幂等记录：
    - businessType = `DEPOSIT`
    - businessKey = `accountNo`
3. 若幂等记录存在：
    - `SUCCESS` -> 根据 requestId 查交易流水并返回首次成功结果
    - `PROCESSING` -> 抛 `IDEMP001`
    - `FAILED` -> 抛 `IDEMP003`
4. 若不存在幂等记录：
    - 调用 `idempotentService.init(requestId, "DEPOSIT", accountNo)`
5. 调用 `accountRepository.lockByAccountNo(accountNo)` 获取账户
6. 账户不存在：
    - `idempotentService.markFailed(...)`
    - 抛 `ACCT404`
7. 若账户状态为 `FROZEN`：
    - `idempotentService.markFailed(...)`
    - 抛 `ACCT005`
8. 若账户状态不是 `ACTIVE`：
    - `idempotentService.markFailed(...)`
    - 抛 `ACCT001`
9. 计算：
    - `balanceAfter = currentBalance.add(amount)`
10. 更新账户余额和 `updatedTime`
11. 生成 `txnId`
12. 保存交易流水：
- `txnType = DEPOSIT`
- `txnStatus = SUCCESS`
- `debitAccountNo = null`
- `creditAccountNo = accountNo`
- `debitBalanceAfter = null`
- `creditBalanceAfter = balanceAfter`
- `amount = request.amount`
- `remark = request.remark`
- `txnTime = now`
13. 调用 `idempotentService.markSuccess(requestId, "DEPOSIT", "B0000", "success")`
14. 返回 `DepositResponse`

### 异常处理
- `BizException`：标记幂等失败后继续抛
- `Exception`：标记幂等失败后包装或继续抛出，由全局异常处理器统一处理

## 9. 首次结果重放规则

当幂等记录状态为 `SUCCESS` 时：
- 使用 `transactionRepository.findByRequestId(requestId)` 查询交易记录
- 构造 `DepositResponse`：

| 字段 | 取值 |
|---|---|
| requestId | transaction.requestId |
| txnId | transaction.txnId |
| accountNo | transaction.creditAccountNo |
| amount | transaction.amount |
| balanceAfter | transaction.creditBalanceAfter |
| txnStatus | transaction.txnStatus |
| txnTime | transaction.txnTime |
